## üîç Research & Learning

### 1. Key Roles in an Agile Team & What They Do

#### **Developers**
- Build features and write code  
- Handle integrations, bug fixes, and technical improvements  
- Collaborate with PM, QA, and UX  
- Estimate effort and break tasks into manageable units  
- Maintain code quality and documentation  

#### **QA (Quality Assurance)**
- Test features to ensure they work correctly  
- Identify bugs and usability issues  
- Verify acceptance criteria and maintain test cases  
- Help improve product stability and reliability  

#### **Product Manager (PM)**
- Defines product vision and priorities  
- Writes clear requirements and user stories  
- Decides what gets built next and why  
- Works closely with developers, UX, and stakeholders  
- Ensures features align with user needs and business goals  

#### **Agile Project Manager (Agile PM) / Delivery Lead**
- Facilitates Agile processes (planning, flow management, retros)  
- Removes blockers and improves team workflow  
- Ensures smooth flow in Kanban boards  
- Helps maintain team alignment and continuous improvement  
- Not always present in small startups (roles can overlap)  

#### **Supporting Roles**
**Stakeholders**  
- Provide direction, goals, and feedback  
- Often executives, customers, or partners  

**UX / UI Designers**  
- Create user flows, interface designs, and prototypes  
- Collaborate with PM and developers to refine functionality  

**Customer Support**  
- Provides real-world insights into user struggles  
- Helps prioritise bug fixes and usability improvements

## 2. How These Roles Collaborate in Kanban vs. Scrum

### **Kanban (used at Focus Bear)**
- Continuous flow, no fixed sprints  
- Roles collaborate fluidly  
- Work moves through stages: *Backlog ‚Üí In Progress ‚Üí Testing ‚Üí Review ‚Üí Done*  
- No strict ceremonies; communication happens as needed  
- WIP limits ensure developers, QA, and PM are aligned  
- PM continuously refines the backlog while developers pull tasks  

### **Scrum**
- More structured with predefined roles  
- Developers commit to sprint goals  
- Scrum Master manages ceremonies and processes  
- QA tests within sprint cycles  
- PM (Product Owner) works ahead to prepare backlog for future sprints  
- Collaboration focuses on sprint planning, review, and daily standups
