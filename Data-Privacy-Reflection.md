## 2. What Types of Data Are Confidential?
At Focus Bear, confidential data includes:

### **User Data**
- Email, account details  
- Focus session timestamps and patterns  
- Usage analytics  
- Any behavioural data relating to ADHD or executive functioning  
- Messages or support conversations  

### **Company Data**
- Internal documentation  
- Project roadmaps and code  
- Security credentials, tokens, API keys  
- Deployment details  
- Internal discussions or strategy  

### **Teammate Data**
- Personal information shared internally  
- Feedback or performance-related notes  
- Internal communication logs  

---

## 3. Best Practices for Handling Confidential Data
- Store all sensitive files in secure, company-approved locations (Google Drive, Notion, GitHub private repos).  
- Never share personal data through unsecured channels.  
- Avoid downloading sensitive files unless necessary.  
- Do not store credentials in plain text or commit them to repos.  
- Use **environment variables**, encryption, and access controls.  
- Only access data if it is required for a task.  
- Log out of shared or public devices.  
- Follow the rule of least privilege.  

---

## 4. How to Respond to a Suspected Data Breach
If I notice unusual activity, accidental exposure, or a potential breach:

1. **Immediately report it** to my supervisor or security contact.  
2. **Document what happened** (time, file involved, platform, any screenshots).  
3. **Revoke access** if credentials were involved (API key refresh, password change).  
4. **Follow the escalation path** provided by Focus Bear.  
5. Never try to hide or minimise the incident ‚Äî transparency helps the team act quickly.

## 3. Common Mistakes Leading to Data Privacy Issues (and How to Avoid Them)
| Mistake | How to Avoid |
|--------|----------------|
| Copy-pasting sensitive data into AI tools | Sanitize or avoid completely |
| Leaving devices unlocked | Enable automatic screen lock |
| Storing confidential info in personal folders | Use company-approved storage only |
| Using weak or reused passwords | Use strong unique passwords + 2FA |
| Sharing screenshots with visible sensitive data | Blur or crop before sharing |
| Committing secrets to Git | Use `.env` and secret managers |
